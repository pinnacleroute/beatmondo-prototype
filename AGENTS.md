# Prototype Instructions

Run the local server yourself and open the preview in the in-app browser. Do not give the user server-start instructions when you can run it.

Before making substantial visual changes, use the Product Design plugin's `get-context` skill when the visual source is unclear or no longer matches the current goal. When the user gives durable prototype-specific design feedback, preferences, or decisions, record them in `AGENTS.md`.

When implementing from a selected generated mock, treat that image as the source of truth for layout, component anatomy, density, spacing, color, typography, visible content, and hierarchy.

Durable beatmondo direction: always write the brand as lowercase "beatmondo" in visible UI and docs, honoring Gary Burke's original brand style. Treat the product as a premium, gated, rights-controlled sync licensing ecosystem - private club + premium catalog + licensing workflow + secure delivery + catalog intelligence - for serious music supervisors, brands, studios, agencies, strategic partners, artists, rightsholders, and global sync opportunities. Keep public browsing selective; full metadata, project tools, licensing workflows, protected WAV masters, stems, and downloads belong to approved/paid buyer tiers. Use Gary Burke red as a strategic brand accent inside a dark cinematic luxury system, with muted brass/gold only as secondary support.

Durable access direction: represent three buyer tiers - Discovery Access, Professional Buyer, and VIP Sync Access. VIP Sync Access should feel meaningfully faster and more curated, with private selections, priority review, pre-approved terms where available, concierge support, fast-track licensing, and premium secure delivery. Simulate authentication, payment, secure links, file delivery, AI insights, and rights document workflows with frontend state and mock data unless explicitly asked to build real backend behavior.

Durable terminology direction: use "Explore Music" for buyer-facing discovery and reserve "Catalog" for admin/internal operations. Use "Request License" for track/project licensing CTAs and "Request Access" only for account/workspace access. Keep availability, workflow status, delivery state, and admin review state distinct; use "Quote Required" for track availability and "Quote Needed" for internal pipeline status. Prefer "protected master audio" in explanatory copy and "WAV master" only when naming the delivered file. Use "Editorial Hub" for the overview content area, with "Stories", "Media Episodes", "Short Sync Clips", "Catalog Highlights", and "VIP Picks" as child/content labels instead of Blog/Podcast naming.

Durable homepage direction: retain the existing beatmondo logo video in the hero. Keep the public homepage selective and conversion-focused rather than exposing every use case, collection, or internal detail; use "Request Access" as the primary account CTA, "Explore Music" for discovery, and "Request License" only at track or project level.

Durable account direction: provide buyer-facing Sign Up / Request Access, Log In, Forgot Password, and social sign-in entry points as polished simulated frontend flows. Keep authentication and social login behavior mocked unless a real backend is explicitly requested.
